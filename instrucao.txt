================================================================================
  InoLabel — Instruções de Uso
================================================================================

ANTES DE COMEÇAR
----------------
O executável é gerado pelo build e fica dentro de uma pasta com todos os
arquivos necessários para rodar:

  Windows:   dist\InoLabel-windows\InoLabel\InoLabel.exe
  Linux:     dist/InoLabel-linux/InoLabel/InoLabel


ESTRUTURA DE ARQUIVOS NECESSÁRIA
---------------------------------
Coloque os seguintes arquivos NA MESMA PASTA que o executável:

  InoLabel.exe  (ou InoLabel no Linux)
  model.pt          ← modelo YOLO treinado (necessário para detecção automática)
  dataset/          ← pasta com suas imagens ou vídeos

  Exemplo (Windows):
    dist\InoLabel-windows\InoLabel\
    ├── InoLabel.exe
    ├── _internal\          ← não mexa aqui (arquivos internos do app)
    ├── model.pt            ← coloque aqui
    └── dataset\            ← coloque aqui
        ├── imagem1.jpg
        ├── imagem2.png
        └── video.mp4

Formatos de imagem aceitos : .jpg  .jpeg  .png  .bmp  .tif  .tiff
Formatos de vídeo aceitos   : .mp4  .avi  .mov  .mkv

Nota: o model.pt é opcional. Sem ele você ainda pode fazer anotação manual,
      mas a detecção automática YOLO não estará disponível.


COMO RODAR — WINDOWS
---------------------
Opção 1 (mais simples):
  Dê duplo clique em InoLabel.exe

Opção 2 (pelo terminal — recomendado para ver erros):
  1. Abra o Prompt de Comando (cmd) ou PowerShell
  2. Navegue até a pasta do executável:
       cd dist\InoLabel-windows\InoLabel
  3. Execute:
       .\InoLabel.exe

O navegador padrão abrirá automaticamente em:
  http://127.0.0.1:8765

Se o navegador não abrir sozinho, acesse esse endereço manualmente.


COMO RODAR — LINUX
------------------
1. Abra um terminal
2. Navegue até a pasta do executável:
     cd dist/InoLabel-linux/InoLabel
3. Execute:
     ./InoLabel

O navegador padrão abrirá automaticamente em:
  http://127.0.0.1:8765

Se o navegador não abrir sozinho, acesse esse endereço manualmente.

Permissão de execução (caso necessário):
  chmod +x InoLabel


COMO PARAR O PROGRAMA
---------------------
Windows:
  - Feche a janela do terminal onde o InoLabel está rodando, OU
  - Pressione Ctrl+C no terminal

Linux:
  - Pressione Ctrl+C no terminal


ONDE FICAM AS ANOTAÇÕES GERADAS
--------------------------------
As anotações e resultados são salvos na pasta "outputs/" criada
AUTOMATICAMENTE ao lado do executável:

  Windows:  dist\InoLabel-windows\InoLabel\outputs\
  Linux:    dist/InoLabel-linux/InoLabel/outputs/

Para salvar em outro lugar, defina a variável de ambiente antes de rodar:

  Windows (PowerShell):
    $env:INOLABEL_OUTPUT_BASE = "C:\meus_dados\saidas"
    .\InoLabel.exe

  Linux (terminal):
    INOLABEL_OUTPUT_BASE=/home/usuario/saidas ./InoLabel


CONFIGURAÇÕES AVANÇADAS (variáveis de ambiente)
------------------------------------------------
  INOLABEL_OUTPUT_BASE    — pasta onde as anotações serão salvas
                            (padrão: outputs/ ao lado do executável)
  INOLABEL_LOCAL_DIR      — pasta para configurações e atalhos de teclado
                            (padrão: .local/ ao lado do executável)
  CONF_THRESHOLD          — limiar de confiança para detecção YOLO
                            (padrão: 0.40, aceita valores entre 0 e 1)
  MANUAL_IOU_THRESHOLD    — limiar IoU para anotação manual
                            (padrão: 0.30)
  INOLABEL_ENV            — defina como "development" para modo dev
                            (ativa reload automático do servidor)


RESOLUÇÃO DE PROBLEMAS
-----------------------
Problema: "A porta 8765 já está em uso"
  Causa:   Outra instância do InoLabel já está rodando em segundo plano.
  Solução: Windows — abra o Gerenciador de Tarefas e encerre "InoLabel".
           Linux   — execute: kill $(lsof -t -i:8765)

Problema: O navegador abre mas a tela fica em branco
  Solução: Aguarde 3-5 segundos e recarregue a página (F5). O servidor
           pode levar alguns instantes para inicializar na primeira vez.

Problema: Detecção automática não funciona / "model.pt not found"
  Solução: Coloque o arquivo model.pt na mesma pasta que o InoLabel.exe.

Problema: Erro ao iniciar sessão / imagens não encontradas
  Solução: Certifique-se de que a pasta dataset/ existe ao lado do
           executável e contém imagens ou vídeos nos formatos aceitos.

Problema: No Linux — erro "libGL" ou "libgthread"
  Solução: Instale as dependências do sistema:
             sudo apt-get install libgl1 libglib2.0-0

Problema: No Windows — aviso "Windows protegeu seu computador"
  Causa:   O executável não tem assinatura digital comercial.
  Solução: Clique em "Mais informações" e depois em "Executar assim mesmo".

Problema: No Linux — "Permission denied" ao tentar rodar
  Solução: chmod +x InoLabel
